# 수치 알고리즘(numeric algorithms)
<algorithm>헤더가 아닌 <numeric>헤더에 별도로 정의
순차열의 모든 원소를 누적하기 위해 for_each(), transform() 알고리즘을 사용할 수 있지만,   
누적값(상태값)을 기억하기 위해 함수자(functor)를 사용해야 하므로 복잡함   
순차열의 모든 원소에 대해 누적을 구하거나, 순서대로 원소에 연산을 적용해야 할 때,   
수치 알고리즘을 사용하면 쉽게 처리 가능함   

- accumulate(b,e,x,f)   
x를 초기값으로, [b,e) 구간의 모든 원소의 합을 반환   
f를 누적에 사용하여 반환   

- inner_product(b,e,b2,x,f)   
x를 초기값으로, [b,e)의 원소와 구간 [b2, b2+(e-b))의 원소 내적을 반환   
사용자 정의 함수 f사용 가능   

- adjacent_difference(b,e,t,f)   
순차열에서 원소 간의 차이를 구할 떄 사용   
[b,e)의 반복자가 p일 때, (*p - *(p-1))연산을 목적지 순차열 [t,p)에 저장     
(첫번째 원소는 그대로 복사됨)   
사용자 정의 함수 f사용 가능   

- partial_sum(b,e,t,f)   
순차열에서 현재 원소까지의 합을 반환   
[b,e)의 현재 원소까지의 합을 목적지 순차열에 저장   
사용자 정의 함수 f사용 가능
